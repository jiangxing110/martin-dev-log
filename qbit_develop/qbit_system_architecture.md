# Qbit 系统架构总览

## 概述

Qbit 是一个面向商户的跨境支付与加密资产管理平台，基于微服务架构构建，涵盖账户、卡、加密货币、全球账户、融资、风控、合规、资产、报表等 31+ 业务模块。本文档提供系统全局视角的架构描述。

## 核心业务流程

### 1. 提币流程 (Crypto Withdrawal)

```
商户发起提币
    ↓
OpenAPI / Merchant Portal
    ↓
① 风控检查 (Drools: 风控规则)
    ↓
② 合规筛查 (Chainalysis AML)
    ↓
③ 费用计算 (Drools: 费用规则)
    ↓
④ 资产冻结 (Assets System)
    ↓
⑤ 支付路由 (Drools: 路由规则 → PayoutFactory)
    ├── OKX 渠道 → OKX SDK
    ├── Gate 渠道 → Gate SDK
    ├── HashKey 渠道 → HashKey SDK
    └── Safeheron 渠道 → Safeheron SDK
    ↓
⑥ 链上广播
    ↓
⑦ 交易确认 → 对账更新 → 通知商户
```

### 2. 出金流程 (Fiat Payout / 全球账户)

```
商户发起出金
    ↓
① 风控检查 (Drools)
    ↓
② 费用计算 (Drools)
    ↓
③ 余额检查 (全球账户余额)
    ↓
④ 支付路由 → PayoutFactory
    ├── 银行渠道 (SWIFT/ACH)
    ├── 支付渠道 (PayPal/Stripe/etc)
    └── 内部转账 (全球账户互转)
    ↓
⑤ 资金调拨 → FundingTransferFactory
    ↓
⑥ 交易确认 → 对账 → 通知
```

### 3. 消费流程 (Quantum Card)

```
持卡人消费 (Visa/MC)
    ↓
① 收单行 → 卡组网络 → Qbit
    ↓
② 风控检查 (Drools: 消费风控)
    ↓
③ 余额检查 (量子卡余额)
    ↓
④ 交易授权/拒绝
    ↓
⑤ 清算结算 (Core Clear)
    ↓
⑥ 对账 → 费用计算 → 通知商户
```

### 4. OpenAPI 流程

```
商户系统
    ↓
① API 签名验证 (Gateway: SignVerifyFilter)
    ↓
② 认证鉴权 (JWT / API Key)
    ↓
③ 限流检查 (Gateway: RateLimitFilter)
    ↓
④ 路由到对应业务模块
    ├── 账户管理 → Account API
    ├── 卡管理 → Card API
    ├── 交易管理 → Transaction API
    ├── 加密货币 → Crypto API
    ├── 全球账户 → Global Account API
    └── Webhook → Webhook API
    ↓
⑤ 业务处理 → 统一响应
```

